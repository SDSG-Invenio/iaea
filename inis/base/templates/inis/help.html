INIS Input Management

The INIS Input Management system is an all-new application developed by the Nuclear Information Section (NIS) at the IAEA to allow Member States to submit bibliographic records (or batches) and full text documents through an easy-to-use web interface. The new application is slated to supersede traditional means of data transfer, e.g. FTP and is the result of INIS Secretariat’s continuous effort to improve services and user’s experience in human-computer interaction.


Accounts and logging-in

NIS staff will create accounts for the ILOs using their email accounts. You will receive an email asking you to reset your passwords. Once you have done this you will be able to log in using your email address as user name.

Submitting files

Click the Upload button to create a new submission draft. You will see a form in the left-hand side of the screen:

  • Upload name: You can modify the Upload name and use any name you can use to identify the submission. Following a naming convention is recommended.

  • Notes: Optionally you can add notes to your submission.

  • Files: Select one or more files. You can add metadata text files in TTF format containing one or many records. You can also add full-text files.

You don’t need to add them one by one. When you click on Choose files you can select multiple files. You can also just drag-and-drop the files into the Files box.

The following restrictions apply:

  • Only '.ttf', '.inp' and '.pdf' are accepted.
  • The max size per file is 2GiB.

You can click Save to save the draft. You will see it in the Unsubmitted section in the right-hand side. Unsubmitted submissions are just drafts for you to work on them. They are not stored in any database. You can create and delete them as you please. When you have finished adding files to the submission, click Submit.

Validation and feedback

Once the submission actually happens, it comes to the Submitted section and the content is validated. This is just a first level of validation and only most obvious errors are checked.

  • If the submission passes the validation, it will be sent to INIS database. You will see a green tick.

  • If the submission does not pass the validation, you will see the reason (the red area in your snapshot). You will also receive a message (see the envelope in the upper left part of your screen) that you can reply and in case you have questions about it. This submission will be simply ignored in all aspects and you should start a new submission. It will be marked with a red danger icon.

The current implementation validates the following aspects:
  1. Full-text file names should match a known TRN which can be found:
    • in batches within the same upload
    • OR in batches in previously accepted submissions
    • OR in INIS database
  2. Batches should contain TTF metadata
    The file should contain at least one 001 tag with a TRN.
  3. Wrong country code in TRN
    The country code of the submitter and the TRN must be the same for every record.
  4. Duplicated TRNs
    The application will reject the submission if a TRN already exists:
    • Within the same submission
    • OR in previously accepted submissions, unless they are change records
    • OR in INIS database, unless they are change records

More about the interface

Top menu
Clicking on both Submit and the IAEA logo will take you to the front page.
User menu
Click on your name on the upper right part of the screen to open the User Menu.
  • Profile displays some information about your account and lets you change your password.
  • Statistics displays a summary of your submissions.
  • Logout
